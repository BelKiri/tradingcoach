# Task 002.1a: SSH key auth setup for Cursor → Hetzner

## Context

Phase 4 of Task 002 (backend deploy execution) requires Cursor Agent to
run commands on Hetzner server via SSH. Currently SSH key auth from
Mr.K's Mac to deploy@46.224.52.73 is NOT configured. Cursor cannot
authenticate.

This is a one-time setup task. After completion, Cursor (and Mr.K) can
SSH without password to the deploy user. Root or other users not in scope.

## Goal

`ssh deploy@46.224.52.73` from Mr.K's Mac (and from Cursor's terminal
which inherits the same SSH config) logs in without password prompt.

## Out of scope

- Hardening sshd_config (disable password auth globally) — separate
  future task
- SSH key for other users on the server
- Multi-machine SSH setup (only Mr.K's current Mac)

## Phased plan

### Phase 1: Generate or reuse SSH key on Mac

1. Check if `~/.ssh/id_ed25519` exists on Mr.K's Mac:
```

ls -la ~/.ssh/id_ed25519 ~/.ssh/id_[ed25519.pub](http://ed25519.pub) 2>/dev/null

```

2. If exists — skip to Phase 2.

3. If not — generate new ed25519 key (no passphrase, simpler for Cursor
   automation):
```

ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -N "" -C "mrk-tradingcoach-deploy"

```

4. Show contents of public key in terminal output:
```

cat ~/.ssh/id_[ed25519.pub](http://ed25519.pub)

```

5. **Stop. Confirm key exists, await user "next".**

### Phase 2: Copy public key to server (USER ACTION REQUIRED)

Cursor cannot ssh to server without auth. Mr.K runs ONE command manually
with password.

1. Output exact command for Mr.K to copy-paste:
```

ssh-copy-id -i ~/.ssh/id_[ed25519.pub](http://ed25519.pub) deploy@46.224.52.73

```

2. Tell Mr.K: "Run this in your terminal. It will prompt for deploy
   user password ONE TIME. After this, password auth is no longer needed."

3. **Stop. Mr.K runs the command, reports success. Then 'next'.**

### Phase 3: Verify and harden SSH config

1. From Cursor terminal:
```

ssh -o BatchMode=yes -o ConnectTimeout=5 deploy@46.224.52.73 "whoami && hostname"

```
   Expected output: `deploy` + server hostname. No password prompt.

2. If fails — STOP, report exact error. Do NOT attempt fixes that
   modify sshd_config or server security settings.

3. Add server to local `~/.ssh/config` for convenience (creates file
   if missing):
```

cat >> ~/.ssh/config <<'EOF'

Host tradingcoach HostName 46.224.52.73 User deploy IdentityFile ~/.ssh/id_ed25519 EOF chmod 600 ~/.ssh/config

```

4. Verify shortcut works:
```

ssh tradingcoach "echo ok"

```
   Expected output: `ok`

5. **Stop. Output Phase 3 summary. Task complete when verification passes.**

## Constraints (mandatory for Cursor)

- Show command before executing
- Do NOT modify server sshd_config or any /etc/ files
- Do NOT touch root user
- Do NOT add multiple keys, only ed25519 generated/found in Phase 1
- If Phase 2 ssh-copy-id is needed, EXPLICITLY tell Mr.K it's a manual
  one-time action with password — do NOT try to automate it
- After Phase 3, the only difference on Mr.K's Mac should be: maybe a
  new ed25519 key pair, an entry in ~/.ssh/config, and authorized_keys
  on server has new line — nothing else

## Never touch (for this task)

- `tradecoach/`, `frontend/`, `tests/`, `v0/`, `docs/`
- Any server files outside `/home/deploy/.ssh/`
- sshd_config or systemd ssh.service
- Existing SSH keys (if `id_ed25519` already exists, reuse, don't regenerate)
